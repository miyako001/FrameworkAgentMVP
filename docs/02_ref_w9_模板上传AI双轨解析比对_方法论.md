# W9 参考记录 - 模板上传 AI 双轨解析比对方法论

创建日期：2026-05-15  
最后更新：2026-05-15  
适用范围：模板上传、字段解析、表格字段识别、AI 比对告警

---

## 1. 目标

在不破坏现有上传主链路的前提下，提升模板解析的稳定性、可解释性和可回溯性。

核心原则：

1. 本地规则解析为主结果。
2. AI 语义解析为辅助比对。
3. 差异以 warning 形式回传，不阻断上传。
4. 识别优先于拒绝，规范化优先于报错。

---

## 2. 方法论总览

```
模板上传
  -> 占位符识别与规范化
  -> 点号字段分层解析
  -> 本地规则解析（主）
  -> AI 语义解析（辅）
  -> 差异比对并生成 warnings
  -> 继续保存（以本地结果为准）
```

---

## 3. 方法论一：占位符语法兼容（先识别，再规范）

要求至少兼容以下写法：

1. {字段名}
2. {{字段名}}
3. [字段名]
4. (字段名)

执行策略：

1. 识别多种包裹符号，不因语法差异直接失败。
2. 统一规范化为内部标准格式（建议 {字段名}）。
3. 规范化行为写入 warnings，供前端提示用户。

---

## 4. 方法论二：点号字段识别（至少两段）

解析规则：

1. 占位符包含点号且段数 >= 2，即作为表格/复合字段候选。
2. 两段写法：{A.B} -> 表名 A，列名 B。
3. 多段写法：{A.B.C} -> 表名 A，列路径 B.C。
4. 无法明确归类时保留候选并写 warning，禁止静默丢弃。

重点兼容目标：

- {Acronym.WordAcronym} 必须可识别。

---

## 5. 方法论三：双轨解析（规则主导，AI 辅助）

推荐流程：

```
本地解析 -> AI 语义解析 -> 差异比对 -> warnings 输出 -> 保存
```

约束：

1. AI 结果不直接覆盖数据库配置。
2. AI 不可用、超时、返回异常都不影响上传成功。
3. 服务层必须支持降级提示：AI 比对未执行，已降级为本地解析。

---

## 6. 方法论四：差异输出标准（统一 warning）

至少输出以下 4 类 warning：

1. 语法规范化提示（如 {{项目名称}} -> {项目名称}）。
2. 本地有 AI 无（可能 AI 漏识别）。
3. AI 有本地无（可能本地规则漏解析）。
4. 点号层级冲突（如 A.B 与 A.B.C）。

建议补充：

- 表格列数量不一致。
- 表格列名集合不一致。

---

## 7. 方法论五：用户沟通模板（统一答复结构）

建议输出结构：

1. 诊断结果
2. 根本原因
3. 建议行动
4. 相关文档

示例：

```
【诊断结果】
模板包含扩展占位符/点号字段，已触发兼容解析与 AI 双轨比对。

【根本原因】
历史解析规则偏向标准 {} 和两段点号字段，导致部分写法漏识别。

【建议行动】
1. 开启 aiVerify=true 获取差异提示。
2. 回显 warnings 中的规范化信息给用户确认。
3. 对 {Acronym.WordAcronym} 这类字段按表格候选处理。

【相关文档】
- docs/06_issue_w8_模板上传AI双轨解析比对.md
- docs/05_api_接口规范.md
```

---

## 8. 落地检查清单

上线前至少确认：

1. aiVerify=false 与历史行为一致。
2. aiVerify=true 且 AI 正常时能返回差异 warnings。
3. AI 不可用时上传仍成功且有降级 warning。
4. 四种占位符写法均可识别并规范化。
5. {Acronym.WordAcronym} 与 {A.B.C} 均可进入候选结果。
