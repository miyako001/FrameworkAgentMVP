# 知识索引（Docs）

本目录按五类前缀组织：

- `01_kb_*`：知识库（概念、需求、技术与功能知识）
- `02_ref_*`：参考记录（周流程、验收、修复、进度）
- `03_guide_*`：操作指南（协作、执行、落地方式）
- `04_arch_*`：架构设计（架构图、模块边界、时序、部署拓扑）
- `05_api_*`：接口规范（接口清单、契约、错误码）

## 命名规范

- 命名格式：`NN_类型_主题.md`
- `NN`：两位数字前缀，保证目录内稳定排序
- `类型`：短标识（如 `kb`、`ref`、`guide`）
- `主题`：尽量使用简洁中文语义，避免空格和特殊符号
- 周期文档建议采用 `wN` 标记，例如：`02_ref_w7_验收报告.md`

## 预留类型（扩展位）

- `04_arch_*`：架构设计（架构图、模块边界、时序、部署拓扑）
- `05_api_*`：接口规范（接口清单、契约、错误码）
- `06_issue_*`：问题追踪（Bug、功能问题、改进需求、技术债务）
- `07_data_*`：数据设计（数据模型、表结构、迁移）
- `08_test_*`：测试资产（测试计划、测试用例、回归报告）
- `09_ops_*`：运维部署（部署手册、环境配置、监控告警）
- `10_sec_*`：安全合规（鉴权、审计、合规检查）
- `11_product_*`：产品设计（PRD、交互稿说明、业务规则）
- `12_module_*`：模块总览（核心模块、职责边界、阅读入口）
- `13_review_*`：知识回顾（阶段总结、经验复盘、行动项）

说明：当前仓库已启用 `01`、`02`、`03`、`04`、`05`、`06`、`12`、`13` 八类（`06_issue_*` 为新启用），其余类型可按需要逐步启用。
