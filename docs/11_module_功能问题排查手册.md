# 功能问题排查手册（11_module）

> 目的：当用户反馈“某功能有问题”时，快速定位“功能在哪里、预期是什么、哪一步不符合预期”。

## 1. 一页式排查流程

1. 明确反馈：用户说了什么，在哪个页面，做了什么操作。
2. 归类功能：模板管理 / 对话填充 / 导入填充 / AI提取 / 文档生成。
3. 对照预期：查本手册对应功能的“预想行为”。
4. 定位步骤：按“链路步骤”逐步核对，找出第一处不符合预期的位置。
5. 收集证据：请求参数、响应结果、日志关键字、数据库状态、文件落盘情况。
6. 快速结论：前端问题 / 接口问题 / 业务逻辑问题 / 数据问题 / 环境问题。

## 2. 功能排查总表（先看这个）

| 功能 | 用户入口 | 关键接口 | 预想结果 | 首查点 |
| --- | --- | --- | --- | --- |
| 模板上传与解析 | 模板管理/上传页面 | `/api/templates` | 返回 `templateId` 与 `parseResult` | 上传文件类型与模板占位符格式 |
| AI对话填充 | 对话填写页面 | `/api/chat/start`、`/api/chat/message`、`/api/chat/message/stream` | 会话可创建，字段可抽取，进度持续增长 | 会话ID、message入参、SSE事件流完整性 |
| 导入填充（规则） | 导入填写页面 | `/api/import/upload`、`/api/import/parse?useAI=false` | 可解析并产生字段映射 | 文件类型识别、解析错误信息 |
| 导入填充（AI） | 导入填写页面 | `/api/import/parse?useAI=true` | AI提取可返回字段并可确认 | AI提取结果是否为空，是否回退规则解析 |
| 文档生成与下载 | 生成入口/下载按钮 | `/api/generate`、`/api/generate/download/{fileName}` | 返回下载地址且文件可打开 | 字段校验结果、输出文件是否存在 |

## 3. 功能分册排查

### 3.1 模板上传与解析

预想行为：

- 上传合法 docx 后，系统应返回 `success=true`。
- 自动解析出普通字段和表格字段。
- 模板可在列表中看到并可下载。

链路步骤（逐步核对）：

1. 前端是否提交了 `multipart/form-data`。
2. 后端是否收到 `file` 与 `name`。
3. 模板文件是否成功落盘到 `storage/templates`。
4. 解析结果是否包含字段或表格定义。
5. 模板元数据是否写入数据库。

常见偏差：

- 用户反馈“上传成功但没有字段”：多为占位符写法不规范。
- 用户反馈“看不到模板”：多为入库失败或列表过滤条件问题。

### 3.2 AI对话填充

预想行为：

- 能创建会话并返回会话ID。
- 每轮消息后进度增加，必要时返回校验错误。
- 流式接口应按顺序返回 `message -> metadata -> done`。

链路步骤（逐步核对）：

1. `/api/chat/start` 是否返回有效 `sessionId`。
2. 消息请求是否带 `sessionId` 与 `message`。
3. AI抽取结果是否为空。
4. 字段校验是否拦截了本轮输入。
5. 会话字段是否成功保存。
6. 完成条件是否满足并可进入生成。

常见偏差：

- 用户反馈“AI不回话”：先查流式连接与事件流是否中断。
- 用户反馈“说了但没记住”：先查抽取结果与保存会话步骤。
- 用户反馈“一直问同一个问题”：先查进度计算与必填字段判定。

### 3.3 导入填充（规则/AI）

预想行为：

- 文件上传后可创建导入会话。
- 解析后返回映射结果，用户可手工修正。
- 可基于映射结果生成文档。

链路步骤（逐步核对）：

1. 上传后是否返回 `sessionId`。
2. `parse` 是否成功执行（规则或AI模式）。
3. 是否存在 `mappings` 数据。
4. 用户修正后映射是否更新成功。
5. 生成是否返回 `downloadUrl`。

常见偏差：

- 用户反馈“AI模式没结果”：先查 AI提取是否为空及回退路径。
- 用户反馈“字段全错位”：先查字段匹配策略与模板字段配置。

### 3.4 文档生成与下载

预想行为：

- 生成接口返回成功和下载地址。
- 下载接口返回可打开的 docx 文件。

链路步骤（逐步核对）：

1. 生成请求中 `templateId` 是否有效。
2. 字段和表格数据是否通过校验。
3. 输出文件是否写入 `storage/output`。
4. 下载路径对应文件是否存在。

常见偏差：

- 用户反馈“生成失败”：优先看 `validationErrors`。
- 用户反馈“下载404”：优先看输出文件是否落盘。

## 4. 证据收集清单（排查必留）

- 用户操作时间点（精确到分钟）。
- 请求路径、方法、请求体（隐去敏感信息）。
- 响应状态码与响应体。
- 对应日志片段（含错误堆栈或 warning）。
- 数据库关键记录（会话、映射、模板）。
- 文件系统状态（templates/uploads/output 是否存在目标文件）。
