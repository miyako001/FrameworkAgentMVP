# 主要模块总览（11_module）

> 项目：通用 Word 智能填充 Agent  
> 用途：按“功能全链路”理解系统，不重复架构分层说明。

## 1. 文档定位

- 本文回答“某个重要功能是如何从创建到使用跑通的”。
- 重点放在业务链路、输入输出、成功判定、失败回退。
- 架构分层请看 04 文档，接口细节请看 05 文档。

## 2. 功能全链路地图

1. 模板创建链路：上传模板 -> 解析字段 -> 配置启用。
2. AI 对话填充链路：创建会话 -> 对话抽取 -> 校验保存 -> 完成生成。
3. 文件导入填充链路：上传文件 -> 规则或 AI 解析 -> 匹配确认 -> 生成。
4. AI 智能提取链路：参考文件解析 -> 批量提取 -> 置信度分级 -> 人工确认 -> 生成。
5. 文档生成闭环：统一生成入口 -> 输出文档 -> 下载交付。

## 3. AI 对话填充全链路（重点）

### 3.1 功能目标

- 用户通过自然语言逐步填写模板字段。
- 系统实时抽取字段并反馈进度。
- 完成后直接触发生成。

### 3.2 创建阶段（能力准备）

1. 配置模板字段：字段名、类型、必填、引导话术。
2. 初始化 AI 能力：字段抽取、快捷指令识别、校验策略。
3. 会话持久化准备：用于断点续聊和状态恢复。

### 3.3 使用阶段（运行流程）

1. 启动会话：用户选择模板，系统返回会话 ID。
2. 发送消息：用户输入自然语言。
3. 字段抽取：系统从本轮消息提取字段值。
4. 格式校验：按字段类型验证（电话、日期、数字等）。
5. 保存进度：写入会话状态并更新已采集字段。
6. 继续引导：系统询问下一未完成字段。
7. 完成判定：必填字段收集完成后标记会话完成。
8. 生成文档：进入生成链路并返回下载地址。

### 3.4 输出结果

- 对话文本反馈。
- 本轮抽取字段与校验结果。
- 会话进度（0~1）。
- 完成状态与可下载文档地址。

### 3.5 失败回退

- 抽取失败：提示用户改写输入或分条输入。
- 校验失败：返回具体字段错误并要求重填。
- 会话丢失：重新创建会话并恢复上次已保存数据（如可恢复）。

## 4. AI 智能提取全链路（重点）

### 4.1 功能目标

- 从 PDF/Word/Excel/图片等参考资料中，自动批量提取字段。
- 通过置信度分级保障可用性与可控性。

### 4.2 创建阶段（能力准备）

1. 文件解析能力准备：文本、表格、图片 OCR。
2. 批量提取 Prompt 与字段映射规则准备。
3. 多引擎策略准备：主引擎与兜底引擎。

### 4.3 使用阶段（运行流程）

1. 上传参考文件并创建提取会话。
2. 解析文件内容（文本、表格、图片）。
3. 批量提取目标字段。
4. 字段匹配（精确/模糊/语义）。
5. 计算最终置信度并分级（高/中/低）。
6. 人工确认低置信度字段。
7. 进入文档生成闭环。

### 4.4 输出结果

- 字段值清单。
- 置信度与匹配方式。
- 需人工确认项。
- 可生成文档的数据集。

### 4.5 失败回退

- AI 提取为空：回退规则解析路径。
- 引擎不可用：自动切换备用引擎。
- 图片 OCR 受限：提示用户手工补录关键字段。

## 5. 导入填充全链路

### 5.1 功能目标

- 用结构化文件快速填充模板，减少手工输入。

### 5.2 运行流程

1. 上传导入文件并创建导入会话。
2. 执行解析（规则模式）或 AI 解析（AI 模式）。
3. 自动匹配模板字段。
4. 用户手工调整映射。
5. 生成文档并下载。

### 5.3 成功判定

- 导入会话存在。
- 映射可确认。
- 生成接口返回可下载地址。

## 6. 模板管理全链路

### 6.1 功能目标

- 形成可复用、可配置、可启停的模板资产。

### 6.2 运行流程

1. 上传模板文件。
2. 自动解析普通字段与表格字段。
3. 管理员调整字段配置（必填、类型、提示词）。
4. 设置模板启用状态。
5. 在对话/导入/生成链路中复用。

## 7. 文档生成全链路

### 7.1 功能目标

- 提供统一输出能力，支持多输入来源。

### 7.2 输入来源

- 手动填报字段。
- 对话填充字段。
- 导入或 AI 提取后的字段与表格数据。

### 7.3 运行流程

1. 校验模板与字段合法性。
2. 替换普通占位符。
3. 填充表格行数据。
4. 输出 docx 并返回下载路径。

## 8. 关键验收点（功能视角）

- AI 对话链路：从创建会话到生成文档可完整闭环。
- AI 提取链路：低置信度字段必须可人工确认。
- 导入链路：规则与 AI 两种解析模式均可用。
- 模板链路：上传后字段可被正确解析与管理。
- 生成链路：输出文档内容与输入字段一致。
