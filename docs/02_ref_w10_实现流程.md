# W10 实现流程 - Microsoft Agent Framework 真实落地

**周期**：第 10 周  
**里程碑**：M10 - Agent 架构升级  
**目标**：将现有"Prompt 工程 + 硬编码工具选择"架构，升级为基于 `Microsoft.Agents.AI` 的真实 Agent + 工具调用体系，使 AI 自主决策调用哪个工具、何时调用

**⚠️ 核心功能说明**：
- 本周是架构升级周，不新增业务功能，仅改造 AI 决策层
- `Microsoft.Agents.AI 1.5.0` 包已在 `.csproj` 引用，本周正式启用
- 现有 `Tools/`、`Services/`、`Agents/` 代码原地保留，新增包装层衔接
- 改造遵循最小侵入原则：ChatController / ImportController 接口契约不变
- 降级保障：Agent 调用失败时自动回退到原有硬编码路径

### 2.2 FallbackChatClient 设计

新建 `Services/FallbackChatClient.cs`，实现 `IChatClient` 接口，内部复用 `MultiEngineLLMService` 的降级链：

```csharp
using Microsoft.Extensions.AI;

namespace FrameAgentWordFill.Services;

/// <summary>
/// 将 MultiEngineLLMService 包装为标准 IChatClient。
/// 降级顺序：Copilot SDK → Azure OpenAI → OpenAI → 本地规则。
/// </summary>
public sealed class FallbackChatClient : IChatClient
{
    private readonly MultiEngineLLMService _multiEngine;
    private readonly ILogger<FallbackChatClient> _logger;

    public FallbackChatClient(MultiEngineLLMService multiEngine, ILogger<FallbackChatClient> logger)
    {
        _multiEngine = multiEngine;
        _logger = logger;
    }

    public async Task<ChatResponse> GetResponseAsync(
        IList<ChatMessage> messages,
        ChatOptions? options = null,
        CancellationToken cancellationToken = default)
    {
        // 将 messages 拼合为单一 prompt（保留历史上下文）
        var prompt = BuildPromptFromMessages(messages);
        var (response, engine) = await _multiEngine.CallWithFallbackAsync(prompt, cancellationToken);

        _logger.LogDebug("FallbackChatClient 使用引擎: {Engine}", engine);

        return new ChatResponse(
            new ChatMessage(ChatRole.Assistant, response)
        );
    }

    public IAsyncEnumerable<ChatResponseUpdate> GetStreamingResponseAsync(
        IList<ChatMessage> messages,
        ChatOptions? options = null,
        CancellationToken cancellationToken = default)
        => throw new NotSupportedException("流式输出暂不支持，请使用 GetResponseAsync。");

    public ChatClientMetadata Metadata => new("FallbackChatClient", null, null);
    public TService? GetService<TService>(object? key = null) where TService : class => null;
    public void Dispose() { }

    private static string BuildPromptFromMessages(IList<ChatMessage> messages)
    {
        // system prompt 置顶，其余按顺序追加
        var parts = messages.Select(m => $"[{m.Role}]\n{m.Text}");
        return string.Join("\n\n", parts);
    }
}
```

### 3.2 WordFillPlugin（对话填充场景）

```csharp
using System.ComponentModel;
using FrameAgentWordFill.Models.Templates;
using FrameAgentWordFill.Tools;
using System.Text.Json;

namespace FrameAgentWordFill.Plugins;

/// <summary>
/// 对话填充场景工具集，由 WordFillAgent 按需调用。
/// </summary>
public sealed class WordFillPlugin
{
    private readonly AIFieldExtractor _fieldExtractor;
    private readonly DataValidator _dataValidator;

    public WordFillPlugin(AIFieldExtractor fieldExtractor, DataValidator dataValidator)
    {
        _fieldExtractor = fieldExtractor;
        _dataValidator = dataValidator;
    }

    [Description("从用户自然语言消息中提取字段值。返回 JSON 格式的字段名到值的映射。")]
    public async Task<string> ExtractFieldsFromMessageAsync(
        [Description("用户输入的原始消息")] string userMessage,
        [Description("需要提取的字段列表，JSON 数组，每项包含 name/fieldType/required")] string fieldsJson)
    {
        var fields = JsonSerializer.Deserialize<List<Field>>(fieldsJson,
            new JsonSerializerOptions { PropertyNameCaseInsensitive = true })
            ?? new List<Field>();

        var result = await _fieldExtractor.ExtractFieldsAsync(userMessage, fields);
        return JsonSerializer.Serialize(result);
    }

    [Description("检测用户消息中是否含有快捷指令（如"全部跳过"、"重新开始"）。返回 JSON，包含 hasShortcut 和 commandType。")]
    public async Task<string> DetectShortcutCommandAsync(
        [Description("用户输入的消息")] string userMessage)
    {
        var result = await _fieldExtractor.DetectShortcutAsync(userMessage);
        return JsonSerializer.Serialize(result);
    }

    [Description("验证单个字段值是否符合模板规则（类型、必填、格式约束）。返回 isValid 和 errorMessage。")]
    public string ValidateFieldValue(
        [Description("字段定义，JSON 格式，包含 name/fieldType/required/validationRule")] string fieldJson,
        [Description("待验证的字段值")] string value)
    {
        var field = JsonSerializer.Deserialize<Field>(fieldJson,
            new JsonSerializerOptions { PropertyNameCaseInsensitive = true });
        if (field == null) return JsonSerializer.Serialize(new { isValid = false, errorMessage = "字段定义无效" });

        var (isValid, errorMessage) = _dataValidator.ValidateField(field, value);
        return JsonSerializer.Serialize(new { isValid, errorMessage });
    }
}
```

### 3.3 ImportPlugin（文件导入场景）

```csharp
using System.ComponentModel;
using FrameAgentWordFill.Models.Templates;
using FrameAgentWordFill.Models.Parsing;
using FrameAgentWordFill.Tools;
using System.Text.Json;

namespace FrameAgentWordFill.Plugins;

/// <summary>
/// 文件导入场景工具集，由 ImportAgent 按需调用。
/// </summary>
public sealed class ImportPlugin
{
    private readonly AIBatchExtractor _batchExtractor;
    private readonly FieldMatcher _fieldMatcher;

    public ImportPlugin(AIBatchExtractor batchExtractor, FieldMatcher fieldMatcher)
    {
        _batchExtractor = batchExtractor;
        _fieldMatcher = fieldMatcher;
    }

    [Description("从已解析的文档内容中批量提取字段值。输入文档摘要与模板字段列表，返回 AI 提取结果 JSON，含 fieldName/fieldValue/confidence/matchMethod。")]
    public async Task<string> BatchExtractFieldsAsync(
        [Description("已解析的文档内容，JSON 格式（ParsedDocumentContent）")] string documentContentJson,
        [Description("模板字段列表，JSON 数组（Field[]）")] string templateFieldsJson)
    {
        var documentContent = JsonSerializer.Deserialize<ParsedDocumentContent>(documentContentJson,
            new JsonSerializerOptions { PropertyNameCaseInsensitive = true });
        var templateFields = JsonSerializer.Deserialize<List<Field>>(templateFieldsJson,
            new JsonSerializerOptions { PropertyNameCaseInsensitive = true })
            ?? new List<Field>();

        if (documentContent == null)
            return JsonSerializer.Serialize(new { error = "文档内容解析失败" });

        var (fields, engine) = await _batchExtractor.ExtractFieldsAsync(documentContent, templateFields);
        return JsonSerializer.Serialize(new { fields, engine });
    }

    [Description("对 AI 提取结果与模板字段进行语义匹配，处理字段别名、同义词和近义映射。返回匹配后的字段赋值建议。")]
    public async Task<string> MatchExtractedFieldsAsync(
        [Description("AI 提取结果 JSON")] string extractedFieldsJson,
        [Description("模板字段列表 JSON")] string templateFieldsJson)
    {
        var extracted = JsonSerializer.Deserialize<List<FrameAgentWordFill.Models.AIExtraction.AIExtractedField>>(
            extractedFieldsJson, new JsonSerializerOptions { PropertyNameCaseInsensitive = true })
            ?? new();
        var templateFields = JsonSerializer.Deserialize<List<Field>>(
            templateFieldsJson, new JsonSerializerOptions { PropertyNameCaseInsensitive = true })
            ?? new();

        var result = await _fieldMatcher.MatchAsync(extracted, templateFields);
        return JsonSerializer.Serialize(result);
    }
}
```

### 4.1 目标

用 `AIAgentBuilder` 组装一个对话填充 Agent，注册 `WordFillPlugin` 的三个方法为工具，

目标文件（新建）：

### 4.2 WordFillAgentFactory

```csharp
using Microsoft.Agents.AI;
using Microsoft.Extensions.AI;
using FrameAgentWordFill.Plugins;

namespace FrameAgentWordFill.Agents;

/// <summary>
/// 对话填充 Agent 工厂。
/// </summary>
public static class WordFillAgentFactory
{
    public static AIAgent Create(IChatClient chatClient, WordFillPlugin plugin)
    {
        return new AIAgentBuilder()
            .WithInstructions(SystemPrompt)
            .WithTool(AIFunctionFactory.Create(
                plugin.ExtractFieldsFromMessageAsync,
                "ExtractFields",
                "从用户自然语言消息中提取字段值"))
            .WithTool(AIFunctionFactory.Create(
                plugin.DetectShortcutCommandAsync,
                "DetectShortcut",
                "检测用户消息中的快捷指令"))
            .WithTool(AIFunctionFactory.Create(
                plugin.ValidateFieldValue,
                "ValidateField",
                "验证字段值是否符合模板规则"))
            .WithChatClient(chatClient)
            .Build();
    }

    // Agents/WordFillAgentConfig.cs 中的常量晋升为 Agent 系统提示词
    private const string SystemPrompt = """
        你是一个专业的 Word 文档填充助手。
        
        你的职责：
        1. 引导用户逐步填写 Word 模板中的所有字段
        2. 从用户的自然语言消息中提取字段值（调用 ExtractFields 工具）
        3. 检测用户是否发出快捷指令，如"跳过"、"重新开始"（调用 DetectShortcut 工具）
        4. 对提取的字段值进行验证（调用 ValidateField 工具）
        5. 字段收集完毕后，主动告知用户可以生成文档
        
        规则：
        - 每次只聚焦一到两个字段，避免一次性提问过多
        - 对用户的回答保持友好和耐心
        - 验证不通过时给出清晰的纠正提示
        - 不要猜测未提及的字段值，必须由用户明确提供
        """;
}
```

using Microsoft.Agents.AI;
using Microsoft.Extensions.AI;
using FrameAgentWordFill.Plugins;

namespace FrameAgentWordFill.Agents;

/// <summary>
/// 文件导入 Agent 工厂。
/// </summary>
public static class ImportAgentFactory
{
    public static AIAgent Create(IChatClient chatClient, ImportPlugin plugin)
    {
        return new AIAgentBuilder()
            .WithInstructions(SystemPrompt)
            .WithTool(AIFunctionFactory.Create(
                plugin.BatchExtractFieldsAsync,
                "BatchExtractFields",
                "从文档内容中批量提取所有模板字段的值"))
            .WithTool(AIFunctionFactory.Create(
                plugin.MatchExtractedFieldsAsync,
                "MatchFields",
                "对提取结果与模板字段做语义匹配，处理别名和同义词"))
            .WithChatClient(chatClient)
            .Build();
    }

    private const string SystemPrompt = """
        你是一个专业的文档字段提取助手。
        
        你的职责：
        1. 接收用户上传的文档内容（Excel/JSON/Word/PDF/图片等）
        2. 调用 BatchExtractFields 工具，从文档中批量提取模板所需字段

        - 无法提取的字段输出 null，并说明原因
        - 不得凭空捏造字段值，不确定时置信度打低分
        - 保持返回格式结构化，方便前端展示

## 步骤 6：改造 ChatService / ImportService 调用方式

### 6.1 原则

- ChatController / ImportController 接口签名**不变**
- Service 层内部将调用入口从"直调 Tool"切换为"调 Agent.RunAsync"
- 原有 `AIFieldExtractor`、`AIBatchExtractor` 保留注册，作为 Plugin 内部依赖（不废弃）

### 6.2 ChatService 改造点

目标文件：`backend/FrameAgentWordFill/Services/ChatService.cs`

改造位置：`ProcessMessageAsync` 方法中的字段提取逻辑

改造前（硬编码工具调用）：
```csharp
var extractedFields = await _fieldExtractor.ExtractFieldsAsync(userMessage, remainingFields);
```

改造后（Agent 决策）：
```csharp
// 构建上下文消息：system 提供当前字段状态，user 提供本轮消息
var messages = new List<ChatMessage>
{
    new(ChatRole.System, BuildFieldContextPrompt(template, session)),
    new(ChatRole.User, userMessage)
};

await using var agentSession = await _wordFillAgent.CreateSessionAsync();
var agentResponse = await _wordFillAgent.RunAsync(agentSession, new AgentRunOptions
{
    Messages = messages
});

// 从 Agent 响应中解析最终字段结果（Agent 已自主调用工具并得出结论）
var extractedFields = ParseAgentFieldResult(agentResponse);
```

新增私有方法：

1. `BuildFieldContextPrompt(template, session)`：将当前已收集/未收集字段状态注入 system 消息
2. `ParseAgentFieldResult(agentResponse)`：从 Agent 最终输出中提取字段 JSON

**降级保障**：

```csharp
catch (Exception ex)
{
    _logger.LogWarning(ex, "Agent 调用失败，降级到直接 Tool 调用");
    extractedFields = await _fieldExtractor.ExtractFieldsAsync(userMessage, remainingFields);
}
```

### 6.3 ImportService 改造点

目标文件：`backend/FrameAgentWordFill/Services/ImportService.cs`

改造位置：`RunAiExtractionAsync`（或等效的 AI 提取入口方法）

改造前：

## 步骤 7：回归验证与降级验证

### 7.1 功能回归用例

**对话模式（WordFillAgent）**：

| 用例 | 验证点 |
|---|---|
| 单字段自然语言输入 | Agent 调用 ExtractFields 正确提取 |
| 快捷指令"跳过此字段" | Agent 调用 DetectShortcut 正确识别 |
| 字段值格式错误 | Agent 调用 ValidateField 并给出纠正提示 |
| 多字段一次性提供 | Agent 自主拆分为多次 ExtractFields 调用 |

**导入模式（ImportAgent）**：

| 用例 | 验证点 |
|---|---|
| Excel 导入标准字段 | Agent 调用 BatchExtractFields 并匹配成功 |
| PDF 含语义相近字段名 | Agent 调用 MatchFields 做语义匹配 |
| 低置信度字段 | 返回结果中标记"需人工确认" |

### 7.2 降级验证用例

| 场景 | 预期行为 |
|---|---|
| IChatClient 全部引擎不可用 | 捕获异常 → 降级到直接 Tool 调用 → 接口成功返回 |
| Agent 返回格式非预期 | ParseAgentFieldResult 捕获解析异常 → 返回空结果 + warning |
| Agent 调用超时 | CancellationToken 超时触发 → 降级 → 不影响主链路 |

### 7.3 对比验证方法

在降级路径和 Agent 路径之间，用同一组测试数据跑两次，对比字段提取准确率与置信度分布，作为 W10 验收依据之一。
